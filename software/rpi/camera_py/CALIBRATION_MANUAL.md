# ボール検出キャリブレーションマニュアル

## 概要

`manual_calibration.py`は、カメラとボール検出アルゴリズムのキャリブレーションデータを収集するためのGUIアプリケーションです。実際のボール位置と検出結果を記録し、距離・角度の推定精度を向上させるためのデータを蓄積します。

## 動作環境

- Raspberry Pi 5
- Picamera2対応カメラ
- Python 3.x
- 必要なライブラリ：tkinter, cv2, numpy, PIL, picamera2
- アームコントローラー（オプション）

- 距離と角度の初期値設定（距離: 100mm、角度: 0.0πrad）

## 操作手順

### 基本的なキャリブレーションフロー

1. **ボールの配置**
   - カメラの前にボールを配置します
   - メジャーなどで正確な距離と角度を測定します

2. **距離・角度の入力**
   - 「距離 (mm)」: ロボット中心からボール中心までの距離をミリメートルで入力
   - 「角度 (πrad)」: ロボット正面からの角度をπラジアン単位で入力
     - 例：正面 = 0.0、右30度 ≈ 0.1667、右90度 = 0.5
   - 初回起動時は自動で「100mm」「0.0πrad」が入力されています
   - 値は手動で編集可能です

3. **計測開始**
   - 「計測開始」ボタンをクリック
   - アプリが自動的に10回ボールを検出します
   - 計測進捗が「0 / 10」から「10 / 10」まで更新されます

4. **結果の確認**
   - 計測完了後、以下の中央値が表示されます：
     - **検出直径 (px)**: 画像上のボール直径（ピクセル）
     - **位置X (px)**: ボール中心のX座標（ピクセル）
     - **位置Y (px)**: ボール中心のY座標（ピクセル）
     - **計算角度 (πrad)**: アルゴリズムが計算した角度
     - **計算距離 (mm)**: アルゴリズムが計算した距離

5. **データ保存**
   - 結果が妥当であれば「OK - CSVに保存」ボタンをクリック
   - データが`calibration_data.csv`に追記されます
   - 保存後、次の計測値が自動的に入力されます：
     - 通常：距離 +20mm、角度は同じ
     - 距離が260mm以上の場合：距離を100mmに戻し、角度 +0.025πrad

6. **繰り返し**
   - ボールを新しい位置に移動
   - 手順2に戻る

### 計測のキャンセル

計測結果が不適切な場合：
- 「キャンセル」ボタンをクリック
- 計測データがリセットされます
- 入力値はそのまま残ります

## 角度の単位について

このアプリケーションでは角度を**πラジアン**単位で扱います。

### πラジアンとは

- 180度 = 1π ラジアン = 1.0 πrad
- 90度 = 0.5π ラジアン = 0.5 πrad
- 30度 = π/6 ラジアン ≈ 0.1667 πrad
- -90度 = -0.5π ラジアン = -0.5 πrad

### 変換式

```
度 → πラジアン: πrad = 度 / 180
πラジアン → 度: 度 = πrad × 180
```

### 例

| 度 | πラジアン |
|----|----------|
| 0° | 0.0000 |
| 15° | 0.0833 |
| 30° | 0.1667 |
| 45° | 0.2500 |
| 90° | 0.5000 |
| -30° | -0.1667 |
| -90° | -0.5000 |

## 自動値更新機能

効率的なデータ収集のため、CSV保存後に次の計測値が自動設定されます。

### 更新ルール

1. **通常の場合**
   - 距離：現在値 + 20mm
   - 角度：変更なし
   - 例：100mm → 120mm → 140mm → ... → 260mm

2. **距離が260mm以上に達した場合**
   - 距離：100mmにリセット
   - 角度：現在値 + 0.025πrad（約4.5度）
   - 例：260mm, 0.0πrad → 100mm, 0.025πrad

### キャリブレーションパターン例

```
角度 0.000πrad: 100, 120, 140, 160, 180, 200, 220, 240, 260mm
角度 0.025πrad: 100, 120, 140, 160, 180, 200, 220, 240, 260mm
角度 0.050πrad: 100, 120, 140, 160, 180, 200, 220, 240, 260mm
...
```

## トラブルシューティング

### カメラが起動しない

- Picamera2が正しくインストールされているか確認
- カメラケーブルの接続を確認
- 他のアプリケーションがカメラを使用していないか確認

### ボールが検出されない

- ボールの色がHSV設定範囲内か確認
- 照明条件を調整
- ボールとの距離が適切か確認（近すぎ・遠すぎないか）
- `ball_detect.py`のHSVパラメータを調整

### アーム初期化エラー

- アームコントローラーの接続を確認（`/dev/ttyAMA0`, `/dev/ttyARM`）
- シリアルポートの権限を確認
- 警告が表示されてもアームなしで続行可能です

### 計測が10回完了しない

- ボールがカメラ視野内にあるか確認
- ボールが他の物体で遮られていないか確認
- 照明条件を改善

## 終了方法

1. 「終了」ボタンをクリック
2. 以下の処理が自動で実行されます：
   - カメラ停止
   - アームサーボのトルクOFF
   - リソース解放

または、ウィンドウの×ボタンで閉じることもできます。

## 推奨キャリブレーション手順

### 1. グリッド配置法

ロボットの前方に格子状にボールを配置してデータを収集します。

**距離範囲**: 100mm ～ 260mm（20mm間隔）  
**角度範囲**: -0.5πrad ～ 0.5πrad（0.025πrad間隔）

合計データ点数：約9距離 × 41角度 = 369点

### 2. データ収集のコツ

- 照明を一定に保つ
- ボールの高さを地面から35mm（ボール半径）に保つ
- 計測ごとにボール位置を正確に測定
- カメラレンズの汚れを事前に清掃

### 3. 品質チェック

- 実測値と計算値の差を確認
- 異常値がないかチェック
- 必要に応じて再計測
